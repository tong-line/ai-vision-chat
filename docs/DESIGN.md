# AI 视觉对话助手 - 设计文档

> 本文档随开发进度持续更新，记录计划 vs 实际实现。

## 1. 项目概述

开发一个 AI 视觉对话助手：用户通过摄像头和麦克风与 AI 交互，AI 能同时理解画面内容和语音/文字输入，并给出回复。

## 2. 架构设计

```
浏览器端（端侧）                    云端
┌─────────────────────┐         ┌──────────────┐
│ 摄像头 / 麦克风采集   │         │ 多模态大模型   │
│ 语音识别 STT         │ ──────► │ API          │
│ 视频帧截图 + 压缩    │         └──────────────┘
│ 成本控制模块         │
│ 对话 UI + TTS       │
└─────────────────────┘
         │
         │ proxy 模式（后续扩展）
         ▼
┌─────────────────────┐
│ 自建后端 /api/chat  │  ← 当前未实现，已预留接口
└─────────────────────┘
```

### 技术栈

| 模块 | 选型 | 说明 |
|------|------|------|
| 前端框架 | React + Vite + TypeScript | 组件化、开发快 |
| 媒体采集 | getUserMedia | 浏览器原生 API |
| 语音识别 | Web Speech API | 零成本，Chrome/Edge |
| 语音合成 | Web Speech API | 零成本 |
| 视觉理解 | 多模态 LLM API | 通义千问 VL 等 |
| 后端 | 无（预留 proxy 模式） | 后续可自行补充 |

### 对话服务抽象

通过 `VITE_CHAT_MODE` 切换三种实现：

| 模式 | 说明 | 状态 |
|------|------|------|
| `mock` | 无 API Key，返回模拟回复 | ✅ 已实现 |
| `direct` | 浏览器直连大模型 API | ✅ 已实现 |
| `proxy` | 请求自建后端 `/api/chat` | 🔲 接口已预留 |

## 4. 成本控制策略对比

| 策略 | 设想 | 实际实现 | 预期效果 | 状态 |
|------|------|---------|---------|------|
| 帧采样 | 每 N 秒最多发送一帧 | | 减少 API 调用次数 | 🔲 待做 |
| 静止检测 | 画面无变化时不发图 | | 避免无效视觉请求 | 🔲 待做 |
| 低分辨率截图 | 640×480 + JPEG 0.7 | | 减少 token 消耗 | 🔲 待做 |
| 上下文截断 | 只保留最近 10 轮对话 | | 控制输入 token | 🔲 待做 |
| 按需视觉 | 纯聊天时不带图片 | | 节省多模态费用 | 🔲 待做 |
