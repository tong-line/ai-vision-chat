# AI 视觉对话助手 - 设计文档

> 本文档随开发进度持续更新，记录计划 vs 实际实现。

## 1. 项目概述

开发一个 AI 视觉对话助手：用户通过摄像头和麦克风与 AI 交互，AI 能同时理解画面内容和语音/文字输入，并给出回复。

**目标用户**：需要快速获取眼前事物信息的普通用户（学习、办公、生活场景）。

**核心价值**：无需拍照上传，直接「看到什么问什么」，降低多模态 AI 使用门槛。

## 2. 架构设计

```
浏览器端（端侧）                    云端
┌─────────────────────┐         ┌──────────────┐
│ 摄像头 / 麦克风采集   │         │ 智谱 GLM-4.6V │
│ 语音识别 STT         │ ──────► │ 多模态 API    │
│ 视频帧截图 + 压缩    │         └──────────────┘
│ 成本控制模块         │
│ Siri 式状态 UI + TTS │
└─────────────────────┘
         │
         │ proxy 模式（后续扩展）
         ▼
┌─────────────────────┐
│ 自建后端 /api/chat  │  ← 当前未实现，已预留接口
└─────────────────────┘
```

### 技术栈

| 模块 | 选型 | 说明 |
|------|------|------|
| 前端框架 | React + Vite + TypeScript | 组件化、开发快 |
| 媒体采集 | getUserMedia | 浏览器原生 API |
| 语音识别 | Web Speech API | 零成本，Chrome/Edge |
| 语音合成 | Edge TTS / Web Speech API | Edge 优先，CORS 失败时回退浏览器 |
| 视觉理解 | 智谱 GLM-4.6V 多模态 API | 直连 + 预留 proxy 模式 |
| 后端 | 无（预留 proxy 模式） | 后续可自行补充 |

### 对话服务抽象

通过 `VITE_CHAT_MODE` 切换三种实现：

| 模式 | 说明 | 状态 |
|------|------|------|
| `mock` | 无 API Key，返回模拟回复 | ✅ 已实现 |
| `direct` | 浏览器直连大模型 API | ✅ 已实现 |
| `proxy` | 请求自建后端 `/api/chat` | 🔲 接口已预留 |

### 交互设计说明

未采用电子人/数字人方案，原因：
- 考核重点是多模态视觉理解与端云成本控制，电子人开发成本高且与核心能力无关
- 参考 Siri 交互模式，通过状态指示器（待命/聆听/思考/回复）提供视觉反馈
- 配合实时语音字幕和自动发送，提升语音交互的自然度与流畅性

## 4. 成本控制策略对比

| 策略 | 设想 | 实际实现 | 预期效果 | 状态 |
|------|------|---------|---------|------|
| 帧采样 | 每 N 秒最多发送一帧 | FrameIntervalGate，默认 3s 间隔 | 减少 API 调用次数 | ✅ 完成 |
| 静止检测 | 画面无变化时不发图 | SceneChangeDetector 32×32 像素指纹对比 | 避免无效视觉请求 | ✅ 完成 |
| 低分辨率截图 | 640×480 + JPEG 0.7 | useFrameCapture 压缩 | 减少 token 消耗 | ✅ 完成 |
| 上下文截断 | 只保留最近 10 轮对话 | directChat trimHistory | 控制输入 token | ✅ 完成 |
| 按需视觉 | 纯聊天时不带图片 | UI 勾选「附带当前画面」 | 节省多模态费用 | ✅ 完成 |

### 成本控制效果估算

假设用户 1 分钟内连续提问 10 次：

| 场景 | 无优化 | 有优化 |
|------|--------|--------|
| 发图次数 | 10 次 | 约 2–4 次（采样+静止检测） |
| 单次图片 token | 高清原图 | 640×480 压缩图 |
| 预估节省 | — | 60–80% 视觉 API 成本 |

## 5. 开发流程说明

项目初期在 `main` 分支直接提交基础功能（commit 保持单一职责）。

明确 PR 要求后，后续功能均通过 feature 分支 + Pull Request 合并：

| PR | 分支 | 内容 |
|----|------|------|
| #1 | `feat/frame-sampling` | 帧采样成本控制 |
| #2 | `feat/scene-static-detect` | 静止画面检测 |
| #3 | `feat/edge-tts` | Edge TTS 语音合成 |
| #4 | `docs/complete-design-doc` | 完善设计文档 |

## 7. 未完成项及原因

| 项目 | 原因 |
|------|------|
| 自建后端 proxy 服务 | 时间优先保证前端主链路，已预留接口 |
| 端到端语音对话（Realtime API） | 成本高、与截图场景不匹配 |
| 电子人/数字人 | 投入产出比低，已用 Siri 式交互替代 |

## 8. 运行说明

```bash
npm install
cp .env.example .env   # 填入智谱 API Key
npm run dev            # http://localhost:5173
npm run build          # 生产构建
```

**演示步骤**：
1. 开启摄像头
2. 点击麦克风说话或输入文字
3. 勾选「附带当前画面」让 AI 看到当前场景
4. 观察状态指示器变化与 AI 回复
